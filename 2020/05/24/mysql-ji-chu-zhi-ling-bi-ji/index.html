mysql基础笔记

No tag
MySQL
Publish Date:   2020-05-24

MySQL基础指令笔记：

若无说明则默认每句代码后面需要加 “;”

登陆与退出:

  1. [数据库文件夹名] -u+用户名 -p+密码(建议-p后先回车再输入密码更为安全)

  2. 使用exit 或者 quit加enter 退出

注释：

  1. 单行注释： – 文字 或者 # 文字

  2. 多行注释：/*文字段*/

操作数据库：

  1. 创建数据库：
    • create database 数据库名
    • create database if not exists 数据库名
    • 可在数据库名称后面增加character set 字符集名称，如character set utf8 或者 character set gbk
  2. 查询：
    1. show databases (所有数据库)
    2. show create database 数据库名称(查看单个数据库)
  3. 修改： 
    * alter database 数据库名称 character set 字符集名称(修改数据库的字符集名称)
  4. 删除：
    1. drop database 数据库名称
    2. drop database if exists 数据库名称
  5. 使用数据库：
    1. use 数据库名称
    2. select database() 查看当前数据库名称
  6. 备份与还原数据库：
    1. 备份：mysqldump -u[username] -p[password] databasename >url[保存路径]
    2. 还原：新建数据库后使用 source url[备份路径]

操作表：

  1. 表的创建(举例演示)： 
create table employee(
    name varchar(), --在括号内输入整数表示字符多少
    id int,
    birthday date, --格式为yyyy-mm-dd
    entrytime datetime(也可使用时间戳timestamp，时间戳默认为当前时间), --格式为yyyy-mm-dd hh:mm:ss

);

最后一个参数后面无需逗号

  1. 表的复制：
    create table 表名 like 需要复制的表名

  2. 查询：

    • 所有表：show tables
    • 单张表: desc 表名

  3. 表的修改：

    1. 表名：alter table 表名 rename to 新的表名
    2. 修改字符集： 同数据库的设置字符集操作
    3. 新增一列：alter table 表名 add 列名 数据类型
    4. 修改列名和数据类型：alter tabe 表名 modify(change) 列名 新列名 新数据类型
    5. 删除列：alter table 表名 drop 列名

  4. 删除表:

    • drop table 表名

    • drop table if exists 表名(与操作数据库时相同，添加if语句更加安全)

操作表中数据:

​ 1. 添加数据：

​ * insert into 表名(列名1，列名2，…)values(数据1，数据2，…); 

insert into student(name,id,gender,age) values ('张三',1,'男',30),('潘阿拉贡','女',16),('狄仁杰','男',23);

  • 注意事项:

  • 列名可只写部分，但是后面的值输入要与列名的数据类型一一对应 。

  • 若表名后不写列名，则默认为所有列

  • 字符类型用’或者”来都可

    1. 删除数据：
      • delete from 表名 [^where 条件]

    [^where 条件]: 可选操作

    • 注意事项：

      1. 不加条件则默认全部删除

    1. 使用truncate table 表名[^效率更高]

      [^效率更高]:先删除原先的表再创建新表

    1. 修改数据：

      • update 表名 set 列名1 = 值1,列名2 = 值2,… [^where 条件]

      [^where 条件]:可选,但若不加条件则默认全部数据都会被更改

查询表数据：

​ 1. select 列名，聚合函数，可多选(也可使用*，代表所有内容) from 表名 条件(可选) 

select * from student; 
--查询student表中的全部数据
select id,age from student; 
--只显示id和age两列数据
select * from student order by id [条件]; 
--默认为按id大小升序排列,条件asc为升序，desc为降序
select * from student order by age asc,id asc; 
--在满足第一个排序条件后再进行第二轮排序(及在学生年龄相同时再通过学号大小进行排序)

  1. 聚合函数(可放在select或者from之后作为条件，聚合函数不计算null值)

    • count()：用于计算数据个数
      • 若想把null值加入可选择ifnull函数：count(ifnull(可能为null的值,想要修改成的值))
    • max()：计算最大值
    • min(): 计算最小值
    • sum(): 计算和
    • avg(): 计算平均值
    1. 条件以及常用的表达式 
    #where
select * from where age >15;
select * from where age != 15;
select * from where age <> 15; 
--与第二句类似
select * from where age >15 &&(and) age<30;
select * from student where age in (13,16,49);
--只查询年龄为这三个数值之一的数据
select * from student where score is not null;
--分数不为空的学生
#注意，null值只可通过 is 或者 is not 来判断
#like(模糊查询)
select * from student where name like '钟%'；
--查询名字开头为钟的数据
select * from student where name like '_钟%';
--查询名字第二个字为钟的数据
select * from student where name like '%钟%';
--查询只要名字中含有钟字的数据
select * from student where age between 10 and 20; 
--同age >=10 and age <= 20
#group by(分组)
select gender,avg(score),count(id) from student group by gender; 
--及将学生按性别分为两行，每行包括性别、平均分、总人数
select gender,avg(score),count(id) from student where score>70 group by sex having count(id)>2;
--与上一行类似，但选择的学生score要大于70，并在分组后的每组人数需大于2
#limit(分页查询)
select * from student limit firstindex,number; 
--从指定的索引firstindex开始展示出number数量的数据

    • 可在from后的列名前加上distinct来去除重复项 

      + select distinct age from student;

    约束:

    1.分类：

    • 主键约束：primary key

      • create table student(
    id int primary key,
    --增加主键约束，id值唯一且非null
    --一张表只允许一列出现主键
    name varchar(20)
)

    • 唯一约束：unique

      • create table student(
    id int unique,
    --id唯一但可以为null
    --可以为多列使用
    name varchar(20) unique
)

    • 非空约束：not null

      • create table student(
    id int,
    name varchar(15) not null
    --学生姓名不能为空
)

    • 外键约束：foreign key

      • create table classroom(
    class_id int constraint for_key foreign key id references student,
    --constraint后为外键名称，可随意命名
    --使教室的id与学生表中的id联系起来

)

    • 增删主键：

      • alter table [表名] drop primary key –delete
      • alter table [表名] modify [列名] [列的数据类型] primary key –add

    • 增删唯一：

      • alter table [表名] drop index [列名]
      • alter table [表名] modify [列名] [列的数据类型] unique

    • 增删非空约束：

      • alter table [表名] modify [列名] [列的数据类型]
      • alter table [表名] modify [列名] [列的数据类型] not null

    • 增删外键：

      • alter table [表名] drop foreign key [外键名称]
      • alter table [表名] add constraint [外键名称] foreign key [列名] references [另表列名]

    • 其他操作：

      • 在外键创建时加入on update cascade 可以让两表的数据同步更新
      • on delete cascade 解除同步更新
      • auto_increment 自动增长，可放在数据类型之后

    事务：

    1. 开启事务：start transaction
    2. 提交：commit
    3. 回滚：rollback
    4. 提交方式：@@autocommit (1为自动，0为手动)
      1. select @@autocommit –查询
      2. set @@autocommit = 0/1 –修改
    5. 隔离等级(从上往下等级依次增强)：
      1. read uncommitted
      2. read committed
      3. repeatable read
      4. serializable
        • 查询：select @@tx_isolation
        • 设置： set global transaction isolation [隔离等级]

    权限管理

    1. 添加用户：create user ‘username’@’localhost’ identified by ‘password’
    2. 删除用户：drop user ‘username’@’localhost’
    3. 修改密码： 
      * update user set password = password('new password') where user = 'username'
* set password for 'username'@'localhost' = password('new password')
    4. 权限查询： show grants for ‘username’@’localhost’
    5. 权限授予：
      • grant [权限] on databasename.listname to ‘username’@’localhost’
      • grant all on . to ‘username’@’localhost’ –add all authority
    6. 权限撤销：
      1. revoke [权限] on databasename.listname from ‘username’@’localhost’
